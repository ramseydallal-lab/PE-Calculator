Major conflicts that still need resolution

1. Final discharge backbone conflict
- Manuscript and supplement currently describe the final discharge backbone as M6:
  - spline wRVU + CPT-3 pooled categories
- PROJECT_STATE also says the locked discharge set is:
  - workrvu_clean (spline) + cpt3_pool
- But saved candidate-testing logs and files on 2026-03-21 are built on "PROC_BEST" with:
  - linear workrvu_clean + cpt3_pool
- This means the current manuscript narrative and at least some saved model artifacts are not yet guaranteed to refer to the same exact discharge specification.
- The March 23 "M6" candidate-additions run does not resolve this conflict because it is a different M6:
  - linear PROC_BEST backbone plus candidate variables
  - not the manuscript-style spline M6 final discharge specification

2. Procedure-signal output conflict
- The file:
  - /Volumes/DallalMinix/ramseymini/Library/CloudStorage/OneDrive-ThomasJeffersonUniversityanditsAffiliates/_Enterprise/stats/Pulmonary Embolism/output/21_proc_signal_compare_2024_RESUMED.csv
  reports AUC values around 0.498 to 0.500 for all models.
- Those values are incompatible with:
  - manuscript claims
  - supplement eTable 5
  - PROJECT_STATE
  - saved sex/steroid comparison outputs
- Interpretation:
  - this file is stale, broken, or based on a flawed evaluation path and should not be treated as source-of-truth for final manuscript numbers.

3. Validation denominator conflict
- Different artifacts currently use three different 2024 denominators:
  - 963,565 = raw 2024 temporal validation cohort
  - 956,434 = candidate-testing / prediction-merge cohort in 25b and 25c
  - 936,327 = complete-case cohort in final QC/performance tables and manuscript
- This is one of the most important unresolved project-state items.
- For the locked pipeline, every comparison and every manuscript number must map explicitly to one of these cohorts, with a documented reason.

4. Modeling-method wording conflict
- Manuscript says:
  - "regularized logistic regression"
- Existing Stata scripts and logs reviewed here show:
  - standard logistic regression via logit/logistic
- Unless a separate regularized modeling step exists elsewhere and is the true final source, the manuscript wording likely needs revision to ordinary logistic regression.

5. Stata-version conflict
- Manuscript says analyses were conducted using Stata 18.
- Locked project instruction requires Stata 19.5.
- This is not a scientific conflict, but it is a methods/reporting conflict that should be harmonized at release.

6. Spline specification conflict
- Supplement/eAppendix describe:
  - restricted cubic spline with 5 knots
  - knots at 5th, 27.5th, 50th, 72.5th, 95th percentiles
- Older do-files reviewed in the parent project often build spline terms using:
  - centile 25, 50, 75
  - mkspline with 4 basis terms
- This needs to be resolved by the final M6 run and then propagated consistently into:
  - manuscript methods
  - supplement methods
  - calculator appendix
  - calculator code
  - locked pipeline scripts

7. CPT pooling threshold conflict
- Supplement section B says:
  - pool CPT-3 families with fewer than 5,000 training cases
- Multiple parent do-files use:
  - POOLN = 20,000
- eAppendix also refers to 41 families meeting a 20,000-case threshold.
- This threshold must be locked once and then made consistent everywhere.

8. Sex-level labeling conflict
- PROJECT_STATE says:
  - sex3 level 3 = male, coefficient about 0.671
  - sex3 level 2 = unknown/other, coefficient about -0.049
- The displayed 25c model output shows:
  - "male" coefficient about -0.046
  - "other/unknown" coefficient about 0.669
- This suggests either:
  - label inversion
  - recoding inconsistency
  - or documentation error
- This matters because the project’s main conclusion is that sex was excluded for lack of predictive gain; the substantive conclusion is probably unchanged, but the table narrative and coefficient labeling must be corrected.

9. Calculator conflict
- Current calculator UI advertises:
  - DISCHARGE: base+PROC_BEST (AUC 0.811)
  - UPDATE: base+major+sentinel (AUC 0.888)
- This does not align with the manuscript scaffold, which currently says:
  - DISCHARGE 0.795
  - UPDATE 0.887
- The calculator appendix also still has major placeholders for:
  - final coefficients
  - spline knots
  - CPT family coefficient table
  - exact sentinel-event values
- Therefore the calculator is still a development scaffold, not yet a release-grade reflection of the manuscript.
